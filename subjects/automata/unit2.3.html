Unit 2.3: Turing Machines

Introduction to Turing Machines (TM)

A Turing Machine (TM) is a theoretical computational model introduced by Alan Turing in 1936. It serves as a foundation for understanding the limits of algorithmic computation. Turing Machines are used to formalize the notion of an algorithm and provide a framework for studying the properties of computation.

A TM accepts Recursive Enumerable Languages, which are generated by Type-0 Grammars in the Chomsky Hierarchy.

Key Features of a Turing Machine

Tape

An infinite tape divided into cells. Each cell can hold one symbol from a finite alphabet, including input symbols and a blank symbol.

Head

A read/write head that moves left or right along the tape, reading the symbol under it and writing new symbols as needed.

States

A finite set of states that determine the machine's behavior at any given time.

Transition Function (Œ¥)

Specifies the machine's behavior based on its current state and the symbol it reads from the tape. It defines the next state, the symbol to write, and the direction to move the head.

Halting State

A state that indicates the computation is complete. Once a halting state is reached, the machine stops.

Formal Definition of a Turing Machine

A Turing Machine is formally defined as a 7-tuple:

Q

A finite set of states.

Œ£ (Sigma)

The input alphabet.

Œì (Gamma)

The tape alphabet, which includes the input alphabet Œ£ and the blank symbol B.

Œ¥ (delta)

The transition function.

q0

The start state.

B

The blank symbol.

F

The set of final or accepting states.

Variations of Turing Machines

While the standard single-tape, single-head Turing Machine is the basis for the theory, several variations exist. Importantly, all these variations are computationally equivalent, meaning they can all compute the same set of functions, though some may be more efficient for specific tasks.

Non-Deterministic Turing Machine (NDTM)

An NDTM can have multiple possible next configurations for a given state and symbol, allowing it to explore multiple computation paths simultaneously.

  • Acceptance: An NDTM accepts an input string if at least one computation path leads to an accepting state.
  • Rejection: An NDTM rejects if all possible computation paths halt and lead to a rejecting state.

Multi-tape Turing Machine

This variant has multiple tapes, each with its own independent read/write head, but is controlled by a single finite control (one state). The input is initially on Tape 1. This can simplify algorithms for tasks like sorting or multiplication.

The transition function for a k-tape machine takes the current state and the symbols under all k heads as input:
\( \delta(q_i, x_1, x_2, ..., x_k) = (q_j, y_1, d_1, y_2, d_2, ..., y_k, d_k) \)

Multi-head Turing Machine (MHTM)

This variant has a single tape but multiple heads that can read, write, and move independently. The transition function considers the current state and the symbols read by all heads.

The transition function for a 2-head machine looks like:
\( \delta(q, a, c) = (p, (X, R), (Y, L)) \)

This means: in state `q`, if head 1 reads `a` and head 2 reads `c`, transition to state `p`, have head 1 write `X` and move Right, and have head 2 write `Y` and move Left.

üìù Quick Review & Memorization

Key Terms & Acronyms

  1. TM (Turing Machine): A theoretical model of computation with an infinite tape, a head, and a set of states.
  2. Tape: An infinite, one-dimensional array of cells used for storage and computation.
  3. Head: A component that can read a symbol from the tape, write a symbol to the tape, and move left or right.
  4. Transition Function (Œ¥): The set of rules that define the TM's behavior.
  5. Halting State: A state from which no further transitions are possible, indicating the end of computation.
  6. Recursive Enumerable Language: The class of languages that can be accepted by a Turing Machine (Type-0 in Chomsky Hierarchy).
  7. DTM (Deterministic TM): A standard TM where each configuration has exactly one possible next configuration.
  8. NDTM (Non-Deterministic TM): A TM that can have multiple possible next moves from a single configuration.
  9. Multi-tape TM: A TM with multiple tapes, each with its own head.
  10. Multi-head TM: A TM with one tape but multiple heads operating on it.

Summary Points

  1. The Turing Machine, introduced by Alan Turing, is the most powerful model of computation and formalizes the concept of an algorithm.
  2. It consists of an infinite tape, a read/write head, and a finite set of states governed by a transition function.
  3. A TM can accept, reject, or loop (run forever) on a given input. It accepts the class of languages known as Recursive Enumerable Languages.
  4. A standard Turing Machine is defined by a 7-tuple: `{ Q, Œ£, Œì, Œ¥, q0, B, F }`.
  5. Variations like Non-Deterministic, Multi-tape, and Multi-head Turing machines exist. While they can make certain algorithms easier to design or more efficient, they are all computationally equivalent to a standard single-tape deterministic Turing Machine.
