Chapter 1: Introduction to Structured Computer Organization

The Concept of a Multilevel Machine

A digital computer is a machine that executes a sequence of instructions, called a program. The most basic instructions a computer's electronic circuits can execute form its machine language. Because these languages are extremely simple and tedious for humans, computers are structured as a sequence of abstractions, or levels. This approach is called structured computer organization.

Each level builds upon the one below it, creating a more powerful and user-friendly environment. The higher levels are more abstract and easier for humans to work with, while the lower levels deal directly with hardware operations.

Languages, Levels, and Virtual Machines

To bridge the gap between human-friendly languages (L1) and machine-friendly languages (L0), two primary methods are used to execute higher-level instructions.

Translation

The entire higher-level program (in language L1) is converted into an equivalent program in the machine's native language (L0). This new L0 program is then executed by the hardware. The program that performs this conversion is called a compiler or assembler.

Interpretation

A special program, called an interpreter, written in L0, takes the L1 program as input. It examines each L1 instruction one by one and executes the equivalent sequence of L0 instructions immediately. No new L0 program is generated.

Virtual Machine

Each level in a computer's hierarchy can be viewed as a virtual machine, with its own unique machine language. For example, a machine M1 with language L1 can be created, even if the physical hardware only understands L0. Programs written for this virtual machine can be either translated or interpreted to run on the real hardware (M0).

A Contemporary Six-Level Machine

Most modern computers can be conceptually divided into six levels, from the physical hardware at the bottom to high-level programming languages at the top.

Level 5: Problem-Oriented Language Level

Consists of high-level languages like C++, Java, or Python, designed for application programmers. Programs are translated by compilers.

Level 4: Assembly Language Level

A symbolic representation of the underlying machine language, making it easier for system programmers to write code. Programs are translated by an assembler.

Level 3: Operating System Machine Level

A hybrid level where new instructions (like system calls for I/O) are added. These new instructions are interpreted by the operating system.

Level 2: Instruction Set Architecture (ISA) Level

This is the language (instruction set) that is visible to an assembly programmer. Instructions are interpreted by the microprogram or executed directly by hardware.

Level 1: Microarchitecture Level

Consists of a collection of registers, an ALU, and a data path. Its operation is controlled by a microprogram or hardware to interpret ISA-level instructions.

Level 0: Digital Logic Level

The true hardware level, composed of gates and circuits that perform simple logical operations like AND and OR.

Evolution of Computer Generations

The history of computing is marked by distinct generations, each defined by a fundamental shift in the underlying technology. This evolution has also blurred the lines between hardware and software.

Zeroth Generation (1642‚Äì1945)

Mechanical Computers. Devices like Pascal's calculator and Babbage's Analytical Engine, which were mechanical and programmable using gears and punched cards.

First Generation (1945‚Äì1955)

Vacuum Tubes. Machines like ENIAC and EDSAC. The concept of the stored-program computer (von Neumann architecture) was established.

Second Generation (1955‚Äì1965)

Transistors. Transistors replaced unreliable vacuum tubes, leading to smaller, faster, and more reliable computers like the PDP-1 and IBM 7094.

Third Generation (1965‚Äì1980)

Integrated Circuits (ICs). Multiple transistors were placed on a single silicon chip, leading to computer families like the IBM System/360 and the popular PDP-11 minicomputer.

Fourth Generation (1980‚ÄìPresent)

Very Large Scale Integration (VLSI). Millions of transistors on a chip enabled the creation of microprocessors and the personal computer (PC) era, dominated by the Intel x86 architecture.

Fifth Generation (Present)

Low-Power and Invisible Computers. A paradigm shift towards ubiquitous and pervasive computing, seen in smartphones, PDAs, and embedded devices.

Technological and Economic Forces

Moore's Law

An observation made by Intel co-founder Gordon Moore, which predicts that the number of transistors on a microchip doubles approximately every 18-24 months. This exponential growth has been the primary driving force behind the rapid increase in computing power and decrease in cost for over four decades.

Key Terms & Acronyms

  1. Program: A sequence of instructions describing how to perform a task.
  2. Machine Language: The set of primitive instructions a computer can directly execute.
  3. Structured Computer Organization: The approach of designing computers as a series of levels or abstractions.
  4. Translation: Converting an entire program from a high-level language to a low-level one before execution.
  5. Interpretation: Executing a high-level program by having a lower-level program examine and carry out each instruction one by one.
  6. Virtual Machine: A hypothetical computer whose machine language is not directly executed by hardware.
  7. ISA (Instruction Set Architecture): The set of instructions and features of a level that are visible to the user of that level.
  8. Microprogram: A program at the microarchitecture level that interprets ISA-level instructions.
  9. Operating System (OS): An interpreter for Level 3 that adds new functionality (like I/O) to the ISA level.
  10. Moore's Law: The observation that the number of transistors on a chip doubles approximately every two years.
  11. Hardware: The physical, tangible components of a computer system (e.g., circuits, memory chips).
  12. Software: The set of algorithms and programs that tell the hardware what to do.

Summary Points

  1. Computers are designed as a series of levels (or virtual machines) to manage complexity, with each level providing an abstraction over the one below it.
  2. High-level languages are made executable through either translation (compiling) or interpretation.
  3. A typical modern computer has six conceptual levels: Digital Logic, Microarchitecture, ISA, Operating System Machine, Assembly Language, and Problem-Oriented Language.
  4. The history of computers is marked by five generations, each defined by a major technological shift: Mechanical, Vacuum Tubes, Transistors, Integrated Circuits, and VLSI/Microprocessors.
  5. The line between hardware and software is arbitrary; any operation performed by software can be built into hardware, and vice versa.
  6. Moore's Law is the key economic and technological force that has driven the exponential growth of computing power for decades.
