Chapter 2: Computer Systems Organization

Processors

A digital computer consists of an interconnected system of three main components: processors, memories, and input/output devices. The processor, or CPU (Central Processing Unit), is the "brain" of the computer, responsible for executing programs stored in main memory.

CPU Components and Organization

Control Unit

Fetches instructions from main memory and determines their type.

Arithmetic Logic Unit (ALU)

Performs operations such as addition and Boolean logic needed to carry out instructions.

Registers

Small, high-speed memory internal to the CPU used to store temporary results and control information.

Key Registers

  • Program Counter (PC): Points to the memory address of the next instruction to be fetched for execution.
  • Instruction Register (IR): Holds the instruction that is currently being executed.

Instruction Execution: The Fetch-Decode-Execute Cycle

The core operation of a CPU is to execute instructions in a sequence known as the fetch-decode-execute cycle:

  1. Fetch the next instruction from memory into the instruction register.
  2. Change the program counter to point to the following instruction.
  3. Determine the type of instruction just fetched.
  4. If the instruction uses a word in memory, determine its location.
  5. Fetch the word, if needed, into a CPU register.
  6. Execute the instruction.
  7. Go back to step 1 to begin the next cycle.

Processor Design Concepts

RISC vs. CISC

CISC (Complex Instruction Set Computer)

Features a large number of complex instructions, some of which may take multiple clock cycles to execute. Often relies on microcode interpretation. Examples: DEC VAX, Intel x86 family.

RISC (Reduced Instruction Set Computer)

Features a small, highly optimized set of simple instructions that can typically be executed in a single clock cycle. Emphasizes direct hardware execution. Examples: SPARC, MIPS.

Design Principles for Modern Computers

  • All common instructions are directly executed by hardware.
  • Maximize the rate at which instructions are issued.
  • Instructions should be easy to decode.
  • Only LOAD and STORE instructions should reference memory.
  • Provide plenty of registers (at least 32).

Parallelism

Instruction-Level Parallelism

  • Pipelining: Divides instruction execution into multiple stages (e.g., fetch, decode, execute). Different instructions can be in different stages simultaneously, increasing throughput. A five-stage pipeline can complete one instruction per clock cycle.
  • Superscalar Architectures: Have multiple execution units, allowing the CPU to issue and execute multiple instructions in parallel during a single clock cycle.

Processor-Level Parallelism

  • SIMD (Single Instruction, Multiple Data): Consists of a large number of identical processors that perform the same sequence of instructions on different sets of data. Common in GPUs.
  • Multiprocessors: Systems with multiple CPUs that share a common memory. They are tightly coupled.
  • Multicomputers: Systems with multiple interconnected computers, each having its own private memory. They are loosely coupled and communicate via message passing.

Primary Memory

The primary memory is where programs and data are stored for access by the CPU. Its basic unit is the bit (a binary digit, 0 or 1). Eight bits form a byte, which is the smallest addressable unit in most modern computers.

Byte Ordering

Big Endian

The "big end" (most significant byte) of a multi-byte word is stored at the lowest memory address. Used by systems like SPARC and IBM mainframes.

Little Endian

The "little end" (least significant byte) of a multi-byte word is stored at the lowest memory address. Used by the Intel x86 family.

Error-Correcting Codes

Extra bits (check bits) are added to memory words to detect or correct errors. The number of bit positions in which two codewords differ is their Hamming distance. To detect 'd' errors, a code needs a distance of d+1. To correct 'd' errors, it needs a distance of 2d+1.

Cache Memory

A small, fast memory placed between the CPU and main memory to reduce the average memory access time. It works based on the locality principle: programs tend to access memory locations that are near recently accessed locations.

  • Hit Ratio (h): The fraction of memory references found in the cache.
  • Miss Ratio: The fraction of memory references not found in the cache (1 - h).
  • Mean Access Time: c + (1 - h)m, where 'c' is cache access time and 'm' is main memory access time.
  • Cache Line: The fixed-size block of data that is transferred between main memory and the cache.

Secondary Memory

Memory Hierarchy

A tiered structure of computer storage. As you move down the hierarchy:

  1. Access time increases.
  2. Storage capacity increases.
  3. Cost per bit decreases.

Magnetic and Solid-State Disks

  • Magnetic Disk: Consists of rotating platters. Key components are tracks (concentric circles), sectors (divisions of a track), and cylinders (a set of tracks at a given radial position).
  • Solid-State Disk (SSD): Uses nonvolatile flash memory. Offers much faster access times than magnetic disks because there are no moving parts (zero seek time).
  • RAID (Redundant Array of Independent Disks): A technology that distributes data across multiple disks to improve performance, reliability, or both. Levels range from RAID 0 (striping for performance) to RAID 5 (striping with distributed parity).

Input/Output (I/O)

I/O equipment is used to transfer information into and out of the computer. Devices are connected to the CPU and memory via one or more buses.

Buses and Controllers

  • Bus: A collection of parallel wires for transmitting address, data, and control signals. Examples include ISA, PCI, and the modern, high-speed PCIe (PCI Express).
  • Controller: Electronics that control an I/O device and handle its access to the bus.
  • DMA (Direct Memory Access): A process where a controller can transfer data directly to or from memory without CPU intervention, improving efficiency.
  • Interrupt: A signal from an I/O controller to the CPU, forcing it to suspend its current program and run an interrupt handler to service the I/O request.

Character Codes

  • ASCII: A 7-bit code for representing characters, suitable for English.
  • Unicode: A 16-bit standard that assigns a unique code point to every character and symbol in the world's languages.
  • UTF-8: A variable-length encoding for Unicode that is backward-compatible with ASCII and is the dominant character set on the web.

Key Terms & Acronyms

  1. CPU: Central Processing Unit, the "brain" of the computer.
  2. ALU: Arithmetic Logic Unit, performs calculations.
  3. PC: Program Counter, holds the address of the next instruction.
  4. IR: Instruction Register, holds the current instruction.
  5. Fetch-Decode-Execute Cycle: The fundamental operation cycle of a CPU.
  6. RISC: Reduced Instruction Set Computer.
  7. CISC: Complex Instruction Set Computer.
  8. Pipelining: A form of instruction-level parallelism that breaks instruction execution into stages.
  9. Superscalar: An architecture that can execute multiple instructions per clock cycle.
  10. SIMD: Single Instruction, Multiple Data; a form of parallel processing.
  11. Big Endian / Little Endian: Systems for ordering bytes in memory.
  12. Cache: A small, fast memory that sits between the CPU and main memory.
  13. Locality Principle: The tendency of programs to reuse data and instructions they have used recently.
  14. RAID: Redundant Array of Independent Disks.
  15. SSD: Solid-State Disk.
  16. Bus: A communication system that transfers data between components.
  17. DMA: Direct Memory Access.
  18. ASCII / Unicode / UTF-8: Standardized character codes.

Summary Points

  1. A computer system is composed of a CPU, memory, and I/O devices interconnected by buses.
  2. The CPU's core function is the fetch-decode-execute cycle, managed by its Control Unit, ALU, and registers.
  3. Performance is enhanced through parallelism, both at the instruction level (pipelining, superscalar) and the processor level (multiprocessors, multicomputers).
  4. Memory is organized in a hierarchy, from fast, small registers and caches to slower, larger main and secondary storage.
  5. The locality principle is the key reason why caches are effective at reducing memory access time.
  6. Secondary storage like SSDs and RAID systems provide persistent, high-capacity data storage.
  7. I/O devices communicate with the CPU and memory through controllers and buses, often using DMA and interrupts to operate efficiently.

Important Enumerations

  1. Fetch-Decode-Execute Cycle Steps:
    • Fetch instruction
    • Increment PC
    • Decode instruction
    • Locate operand(s)
    • Fetch operand(s)
    • Execute instruction
    • Repeat
  2. Memory Hierarchy (Top to Bottom):
    • CPU Registers
    • Cache Memory
    • Primary (Main) Memory
    • Secondary Memory (SSD, Magnetic Disk)
    • Archival Storage (Tape)
  3. RAID Levels (Common):
    • RAID 0: Striping (Performance)
    • RAID 1: Mirroring (Redundancy)
    • RAID 5: Striping with Distributed Parity (Balance)
