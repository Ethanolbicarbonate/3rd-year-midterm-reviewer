Unit 2: User-Centered Design

User-Centered Design (UCD) in Software Engineering

User-Centered Design is a design philosophy and process that places the needs, preferences, and limitations of end users at the forefront of every stage of software development. Its goal is to ensure the final product is not only functional but also intuitive, accessible, and satisfying to use.

Core Principles of UCD

Empathy-Driven Development

Developers strive to understand the user's perspective, challenges, and goals to build solutions that genuinely meet their needs.

Iterative Design

The design process is cyclical. Designs are created, tested with users, refined based on feedback, and tested again until an optimal solution is reached.

User Involvement

End users are actively involved throughout the development process, from requirements gathering to testing, to provide continuous feedback.

Accessibility and Inclusivity

The product is designed to be usable by people with a wide range of abilities and disabilities, ensuring no one is excluded.

What is a User Interface (UI)?

The User Interface (UI) is the visual part of an application through which a user interacts with the system. It focuses not just on usability but also on the overall look, style, and feel of the application.

A popular application often has a UI that is attractive, simple to use, responsive, clear, and consistent across all screens.

Types of User Interfaces

Text-Based User Interface (Command Line Interface - CLI)

Advantages:
  • Offers many and easier customization options.
  • Typically capable of performing more important or complex tasks efficiently.
Disadvantages:
  • Relies heavily on recall (remembering commands) rather than recognition.
  • Navigation can be more difficult for inexperienced users.

Graphical User Interface (GUI)

A GUI uses visual elements like windows, icons, and menus to allow users to interact with the system.

Key Characteristics of a GUI:

  • Windows: Allow different information to be displayed simultaneously.
  • Icons: Represent files, processes, or information visually.
  • Menus: Provide a list of commands to select from, rather than typing them.
  • Pointing: A pointing device (like a mouse) is used to select items.
  • Graphics: Visual elements can be mixed with text on the same display.
Advantages:
  • Requires less expert knowledge to use.
  • Easier to navigate and switch between tasks.
Disadvantages:
  • Typically has fewer options than a CLI.
  • Usually less customizable.

UI Design Principles

1. Structure

Organize the user interface purposefully. This means being consistent, making elements recognizable, grouping related items, and separating unrelated ones. Similar things should look similar.

2. Simplicity

Make simple, common tasks easy to perform. The design should communicate clearly and directly, providing good shortcuts for longer procedures where meaningful.

3. Visibility

Make all required options and materials for a given function visible without distraction. Avoid redundant data and hidden controls for primary tasks.

4. Feedback

Keep users informed of their actions, system status, or errors. This includes confirmation messages, progress indicators, and clear error messages.

5. Tolerance

The design should be flexible and reduce the cost of errors. Allow users to undo and redo actions, and prevent errors by tolerating varied inputs and interpreting all reasonable actions.

üìù Quick Review & Memorization

Key Terms & Acronyms

  1. UCD (User-Centered Design): A design philosophy that prioritizes user needs at every stage of development.
  2. UI (User Interface): The visual part of an application that a user interacts with.
  3. CLI (Command Line Interface): A text-based user interface that relies on typed commands.
  4. GUI (Graphical User Interface): A visual interface that uses windows, icons, and menus.
  5. Empathy: The core UCD principle of understanding and sharing the feelings of the user.
  6. Iteration: The cyclical process of designing, testing, and refining a product based on feedback.
  7. Accessibility: Designing products to be usable by people with disabilities.

Summary Points

  1. User-Centered Design (UCD) is a process focused on user needs, involving empathy, iteration, user participation, and accessibility.
  2. A User Interface (UI) is the visual gateway to an application, with two main types: Text-Based (CLI) and Graphical (GUI).
  3. CLI is powerful and customizable but requires users to recall commands. GUI is intuitive and easy to navigate but can be less flexible.
  4. The five key UI Design Principles are essential for creating effective and user-friendly interfaces.
  5. Structure ensures the UI is organized and consistent.
  6. Simplicity makes common tasks easy and communication clear.
  7. Visibility ensures necessary options are always visible and accessible.
  8. Feedback keeps the user informed about system status and actions.
  9. Tolerance minimizes the cost of user errors through features like "undo" and error prevention.

Important Enumerations

    • Icons
    • Menus
    • Pointing
    • Graphics
