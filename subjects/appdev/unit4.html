Unit 4: Version Control Systems

What is a Version Control System?

Version Control, also known as Source Control or Source Code Management (SCM), is the practice of tracking and managing changes to software code. A Version Control System (VCS) is a software tool that helps teams manage these changes over time.

A VCS keeps track of every modification in a special database. If a mistake is made, developers can "turn back the clock" to compare or revert to earlier versions of the code, minimizing disruption to the team.

Benefits of Version Control Systems

Collaboration

Allows multiple developers to work on the same codebase seamlessly without overriding each other's work.

Backup and Recovery

Enables developers to revert to any older version if something breaks, gets deleted, or an error is introduced.

History Tracking

Provides a complete timeline of the code, showing every change made, by whom, and why (via commit messages).

Branching and Merging

Allows developers to safely experiment with new ideas or fixes in isolation (branching) and then integrate them when ready (merging).

Types of Version Control Systems

Local VCS (LVCS)

Stores all version history on a developer's local computer. It saves different versions of files on the same machine.
Limitation: Collaboration is difficult and prone to error.

Centralized VCS (CVCS)

All files and version history are stored on a single central server. Developers connect to this server to access files.
Limitation: The central server is a single point of failure. If it goes down, no one can collaborate or save changes.

Distributed VCS (DVCS)

Each user has their own full repository and working copy. Changes are saved locally and then "pushed" to a central server to share.
Advantage: No single point of failure. Work can continue even if the central server is offline.

Introduction to Git

Git is a specific, open-source, distributed version control system (DVCS) created by Linus Torvalds in 2005. The entire codebase and its history are available on every developer's computer, making it the go-to VCS for modern software development.

Getting Started with Git

  1. Install Git: Download from git-scm.com and install using the default settings.
  2. Verify Installation: Open a terminal (e.g., cmd, VS Code terminal) and run the command: 
    git --version

Common Git Commands

Basic Configuration

# Initializes a new Git repository in the current directory
git init

# Sets the author name for your commits
git config --global user.name "[Your Name]"

# Sets the author email for your commits
git config --global user.email "[your.email@example.com]"

Core Workflow

# Creates a local copy of a remote repository
git clone [URL]

# Adds changes to the staging area
git add [file]
git add . # Adds all changed files

# Commits staged changes to your local repository with a message
git commit -m "[Descriptive message]"

# Updates your local repository with changes from a remote repository
git pull

# Pushes your local commits to the remote repository
git push

Branching and Merging

# Lists all local branches
git branch

# Switches to a different branch
git checkout [branch-name]

# Combines changes from another branch into your current branch
git merge [branch-name]

History and Status

# Displays the status of your working directory (modified, staged files)
git status

# Shows a history of commits in the current branch
git log

GitHub

GitHub is a for-profit company that offers a cloud-based Git repository hosting service. It provides a web-based graphical interface and access control for Git repositories, making it easier to collaborate and manage projects without needing advanced command-line skills.

Creating a Repository in GitHub

  1. Go to GitHub.com and log in to your account.
  2. Click the "+" icon in the top right corner and select "New repository".
  3. Fill in the details:
    • Repository Name (e.g., my-first-repo)
    • Description (optional)
    • Choose Public or Private
    • Optionally, check "Add a README file"
  4. Click "Create repository". You now have an empty repository hosted on GitHub.

Summary Points

  • A VCS tracks changes to code, allowing for collaboration, history tracking, and recovery.
  • The three main types are Local (LVCS), Centralized (CVCS), and Distributed (DVCS).
  • CVCS has a single point of failure; DVCS does not, as everyone has a full copy of the repository.
  • Git is the most popular, open-source DVCS.
  • The core Git workflow is: `git add` (stage changes), `git commit` (save locally), and `git push` (share remotely).
  • GitHub is a hosting service for remote Git repositories, providing a user-friendly web interface.
