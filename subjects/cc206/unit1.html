Unit 1: Introduction to Application Development

Why Develop Applications?

We develop applications to solve real-world problems, improve efficiency, and provide convenient, accessible, and user-friendly solutions. The ultimate goal is to enhance productivity, communication, and the overall quality of life.

Applications enable task automation, support decision-making, connect people and systems globally, and adapt to evolving technologies to meet both personal and business needs effectively.

Solve Real-World Problems

Applications are created to address specific needs and challenges faced by users or businesses.

Simplify Tasks

By automating processes, applications make complex or repetitive tasks easier and faster to complete.

Drive Innovation

New applications often introduce novel solutions and technologies that push boundaries and create new possibilities.

Application Platforms

Web Applications

A software program that runs in a web browser and is accessed over the internet or a private network. They don't need to be installed on a user's device.

Mobile Applications

A software program designed to run on smartphones, tablets, and other handheld devices. They are installed directly onto the device from an app store.

Desktop Applications

A software program that runs on a desktop or laptop computer's operating system, such as Windows, macOS, or Linux. They are installed locally and can often work offline.

Application Development Approaches

Native Application Development

A native app is developed exclusively for a single platform using a programming language native to that operating system.

  • For Android: Java or Kotlin
  • For iOS: Objective-C or Swift

Native apps acquire all possible advantages of the device and its OS features. They have direct access to hardware like GPS, camera, and microphone, which results in high performance and a better user experience.

Examples: Google Maps, Pinterest, Spotify.

Hybrid Application Development

Hybrid development combines native and web solutions. The core of the app is written using web technologies (HTML, CSS, JavaScript) and then wrapped in a native container.

Plugins like Apache Cordova or Ionic's Capacitor are used to bridge the gap and provide access to native device functionalities. A hybrid app generally consists of two main components: a backend code and a native viewer (essentially a web browser within the app).

Examples: Gmail, Uber, Apple App Store.

Cross-Platform Application Development

This approach allows developers to write code once and reuse it for different platforms. This significantly reduces development time and cost.

Development Tools: Frameworks vs. Kits

Software Development Framework (SDF)

An SDF is a ready-made structure or tool that helps developers build applications faster. It provides reusable parts like login systems and error tracking, so you don't build everything from scratch. It defines the overall flow and architecture of the application.

Analogy: A half-built playground. The swings and slide are already there; you just add your own touches and finish it.

Software Development Kit (SDK)

An SDK is a collection of software development tools in one installable package. It allows for easier integration with existing technologies and provides tools, libraries, and documentation to build applications for a specific platform.

Analogy: A toolbox. It contains all the tools, parts, and instructions you need to build something (like a toy car or a treehouse) from the ground up.

Summary Points

  • Applications are built to solve problems, simplify tasks, and drive innovation.
  • The three main application platforms are Web, Mobile, and Desktop.
  • Native apps are platform-specific (e.g., Kotlin for Android, Swift for iOS) and offer the highest performance.
  • Hybrid apps use web technologies (HTML, CSS, JS) inside a native wrapper to work on multiple platforms.
  • Cross-Platform apps use a single codebase that is compiled or rendered for multiple platforms.
  • An SDK is a toolbox of tools for building, while an SDF is a pre-built structure to build upon.
